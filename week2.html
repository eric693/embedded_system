2.1 — 理論
ARM Cortex-M4 核心架構
流水線、暫存器組、執行模式與特權等級
三級流水線（Pipeline）

Cortex-M4 採用 三級流水線：取指（Fetch）→ 解碼（Decode）→ 執行（Execute）。 在無分支、無資料相依的情況下，每個時脈週期可完成一條指令（CPI ≈ 1）。 當發生 分支（Branch） 時，流水線需要清空（Flush），耗費 1–3 個額外週期； Thumb-2 指令集中的條件執行（IT 指令）可在部分情況下避免分支。

Cortex-M4 支援 硬體乘除法（單週期 32×32 乘法、2–12 週期除法） 以及 FPU（浮點運算單元），支援單精度 IEEE 754 運算， 對 DSP 應用（濾波器、FFT）有顯著加速效果。

暫存器組
通用暫存器（General Purpose Registers） R0 – R3 │ 函式參數 / 回傳值（AAPCS 呼叫慣例） R4 – R11 │ 被呼叫者保存（callee-saved） R12 │ 臨時暫存器（IP） 特殊暫存器（Special Purpose Registers） R13 / SP │ 堆疊指標（MSP / PSP 雙堆疊） R14 / LR │ 連結暫存器（Return Address） R15 / PC │ 程式計數器 狀態暫存器 xPSR │ N Z C V Q GE[3:0] ICI/IT ISR_Number T PRIMASK │ 遮蔽所有可設定優先權的中斷 FAULTMASK │ 遮蔽所有中斷（含 HardFault） BASEPRI │ 遮蔽優先權 ≥ 此值的中斷 CONTROL │ 選擇堆疊指標 / 執行等級
執行模式與特權等級

Cortex-M4 定義兩種 執行模式： Thread Mode（一般應用程式碼）和 Handler Mode（例外處理程式，永遠為特權等級）。 Thread Mode 可以是特權（Privileged）或非特權（Unprivileged）。

模式 觸發條件 特權等級 使用堆疊
Thread Mode Reset 後預設進入 特權 or 非特權（CONTROL[0]） MSP or PSP（CONTROL[1]）
Handler Mode 任何例外（中斷、Fault） 永遠特權 永遠 MSP

使用 RTOS 時，通常將應用任務設為非特權 Thread Mode（使用 PSP）， 核心程式碼使用 MSP，實現記憶體保護（MPU）隔離。

例外與向量表

Cortex-M4 支援最多 240 個外部中斷（IRQ0–IRQ239）加上 16 個內部系統例外。 向量表（Vector Table）存放於 Flash 起始位址 0x08000000， 每個向量為 4 Bytes 的函式指標。Reset 時處理器自動從向量表 offset 0 讀取初始 SP， 從 offset 4 讀取 Reset Handler 並跳轉執行。

c — 向量表結構示意（startup_stm32f446xx.s 對應 C 版本） 
/* 向量表以函式指標陣列形式定義 */
typedef void (*IRQHandler_t)(void);

__attribute__((section(".isr_vector")))
const IRQHandler_t g_pfnVectors[] = {
    (IRQHandler_t)&_estack,   /* 0x00: 初始 MSP 值（來自連結腳本）*/
    Reset_Handler,             /* 0x04: Reset */
    NMI_Handler,               /* 0x08: Non-Maskable Interrupt */
    HardFault_Handler,         /* 0x0C: Hard Fault */
    MemManage_Handler,         /* 0x10: MPU Fault */
    BusFault_Handler,          /* 0x14: Bus Fault */
    UsageFault_Handler,        /* 0x18: Usage Fault */
    0, 0, 0, 0,               /* 0x1C–0x28: 保留 */
    SVC_Handler,               /* 0x2C: SVCall（系統呼叫）*/
    DebugMon_Handler,          /* 0x30: Debug Monitor */
    0,                         /* 0x34: 保留 */
    PendSV_Handler,            /* 0x38: PendSV（RTOS 任務切換用）*/
    SysTick_Handler,           /* 0x3C: SysTick（HAL_Delay 使用）*/
    /* 後續為 STM32F446 的 IRQ0 起 ... */
    WWDG_IRQHandler,           /* IRQ0: 看門狗 */
    /* ... */
};
2.2 — 理論
記憶體架構與位址映射
4 GB 位址空間、記憶體類型、Bus Matrix 與 Bit-Banding
ARM 記憶體映射標準分區

Cortex-M4 採用 統一位址空間（Unified Address Space）， 程式碼、資料與周邊暫存器共用同一個 4 GB 線性位址空間（32-bit）。 ARM 架構對這 4 GB 空間有標準分區建議，各晶片廠商在此框架下實作。

位址 大小 用途 0xFFFFFFFF ┐ │ 512M Vendor System（私有周邊匯流排 PPB） 0xE0000000 ┼ │ 512M 外部裝置（External Device） 0xC0000000 ┼ │ 512M 外部 RAM（External RAM） 0xA0000000 ┼ │ 512M 周邊（Peripheral）─ APB1/APB2/AHB1/AHB2 0x40000000 ┼ │ 512M SRAM（含 Bit-Band Alias 0x22000000） 0x20000000 ┼ │ 512M Code（Flash 0x08000000、Bit-Band Alias） 0x00000000 ┘
STM32F446RE 具體記憶體配置
位址範圍 大小 類型 說明
0x0800 0000 – 0x0807 FFFF 512 KB Flash 程式碼 + const 資料，唯讀（燒錄後）
0x2000 0000 – 0x2001 BFFF 112 KB SRAM1 主要 RAM，堆疊 / 堆積 / .data / .bss
0x2001 C000 – 0x2001 FFFF 16 KB SRAM2 可透過 CCM 存取（高速）
0x4000 0000 – 0x4000 7FFF – APB1 USART2/3、SPI2/3、I2C1/2、TIM2–7 等
0x4001 0000 – 0x4001 5FFF – APB2 USART1/6、SPI1/4、TIM1、ADC1–3 等
0x4002 0000 – 0x4002 7FFF – AHB1 GPIO A–H、DMA1/2、RCC、Flash IF
0x5000 0000 – 0x5006 0BFF – AHB2 USB OTG FS
Bit-Banding（位元帶）

Cortex-M4 的 Bit-Banding 機制允許以 32-bit 寫入的方式原子性地設定或清除單一 bit， 避免 Read-Modify-Write 的 race condition。 SRAM 的 Bit-Band 區域為 0x20000000–0x200FFFFF（1 MB）， 對應的別名區（Alias Region）位於 0x22000000–0x23FFFFFF（32 MB）。

Bit-Band 位址計算公式
設目標 byte 位址為 byte_addr，目標位元為 bit_num（0–7）：
alias_addr = 0x22000000 + (byte_addr − 0x20000000) × 32 + bit_num × 4

對 alias_addr 寫入 1 → 設定該 bit；寫入 0 → 清除該 bit。此操作為硬體保證的原子操作。
Bus Matrix 與等待狀態

STM32F446 內建 多層 AHB Bus Matrix，允許 CPU、DMA1、DMA2 同時存取不同的記憶體匯流排， 在不衝突的情況下實現真正的並行傳輸。Flash 存取速度受 ART Accelerator（指令快取 + 預取）控制， 在 180 MHz 時脈下需設定 5 個等待狀態（Wait States）。

常見錯誤
提升系統時脈後忘記調整 Flash 等待狀態（FLASH_ACR.LATENCY）， 會導致 CPU 從 Flash 讀到錯誤的指令，產生難以重現的隨機崩潰。 應在切換 PLL 之前先設好等待狀態，切換後再確認。
2.3 — 理論 + 實作
GPIO 深度操作
四種工作模式、電氣參數、替代功能與鎖定機制
GPIO 工作模式

STM32F4 系列的每個 GPIO 腳位可獨立設定為以下四種模式（由 GPIOx_MODER 的 2-bit 欄位控制）：

MODER[1:0] 模式 典型應用
00 Input 讀取按鍵、外部訊號
01 General Purpose Output 驅動 LED、控制繼電器
10 Alternate Function UART、SPI、I2C、PWM、ADC
11 Analog ADC 輸入、DAC 輸出（最省電）
輸出類型、速度與上下拉
暫存器 功能 選項
GPIOx_OTYPER 輸出類型 PP（推挽，可主動拉高拉低）/ OD（開汲，需外部上拉，用於 I2C）
GPIOx_OSPEEDR 輸出速度 Low（2 MHz）/ Medium（25 MHz）/ Fast（50 MHz）/ High（100 MHz）
GPIOx_PUPDR 上下拉電阻 No pull / Pull-up / Pull-down（內建約 40 kΩ）
速度設定的影響
輸出速度越高，EMI（電磁干擾）越強、功耗越高。 LED 控制使用 Low 速度即可；SPI 高速傳輸才需要 High 速度。 不必要的高速設定是 EMC 測試失敗的常見原因之一。
替代功能（Alternate Function）設定

當 GPIO 設為 AF 模式時，需額外設定 GPIOx_AFR[L/H] 指定對應的周邊（每腳 4-bit，值 AF0–AF15）。 各腳位與 AF 編號的對應關係必須查閱 Datasheet 的「Alternate function mapping」表格。

c — 設定 PA2 為 USART2_TX（AF7） 
/* 步驟 1：啟用 GPIOA 時脈 */
RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN;

/* 步驟 2：設定 PA2 為 Alternate Function 模式（MODER = 10）*/
GPIOA->MODER &= ~(3U << (2 * 2));
GPIOA->MODER |=  (2U << (2 * 2));

/* 步驟 3：設定推挽輸出、高速、無上下拉 */
GPIOA->OTYPER  &= ~(1U << 2);             /* Push-Pull */
GPIOA->OSPEEDR |=  (3U << (2 * 2));        /* Very High Speed */
GPIOA->PUPDR   &= ~(3U << (2 * 2));        /* No pull */

/* 步驟 4：選擇 AF7（USART2）─ PA2 在 AFRL（pins 0–7）*/
GPIOA->AFR[0] &= ~(0xFU << (2 * 4));       /* 清除 PA2 的 AF 欄位 */
GPIOA->AFR[0] |=  (7U  << (2 * 4));       /* AF7 = USART1/2/3 */
GPIO 鎖定（Lock）機制

GPIOx_LCKR 允許將指定腳位的設定（MODER/OTYPER/OSPEEDR/PUPDR/AFR）鎖定， 鎖定後直到下次系統 Reset 前無法更改。鎖定序列需按 LCKK=1 → LCKK=0 → LCKK=1 → 讀取的特定步驟執行， 常用於安全關鍵系統防止程式錯誤意外改變腳位設定。

2.4 — 理論 + 實作
中斷系統：NVIC 與 EXTI
優先群組、搶佔優先權、子優先權與外部中斷設定
NVIC 優先權模型

Cortex-M4 的 NVIC（Nested Vectored Interrupt Controller） 支援最多 256 個優先權等級（8-bit）。STM32F446 使用高 4-bit（16 個等級）。 優先權分為兩個維度：搶佔優先權（Preemption Priority） 與 子優先權（Sub-Priority），由 AIRCR.PRIGROUP 決定分配方式。

PRIGROUP 搶佔優先權位數 子優先權位數 搶佔等級數 子優先等級數
0 (NVIC_PRIORITYGROUP_4)40161
1 (NVIC_PRIORITYGROUP_3)3182
2 (NVIC_PRIORITYGROUP_2)2244
3 (NVIC_PRIORITYGROUP_1)1328
4 (NVIC_PRIORITYGROUP_0)04116

搶佔優先權較高（數值較小）的中斷可以打斷正在執行的較低搶佔優先權中斷（巢狀中斷）。 子優先權僅在兩個中斷同時發生且搶佔優先權相同時，決定哪個先執行。

FreeRTOS 的要求
使用 FreeRTOS 時必須將 PRIGROUP 設為 NVIC_PRIORITYGROUP_4（全部為搶佔優先權）， 且所有使用 FreeRTOS API（如 xQueueSendFromISR）的 ISR 優先權數值 必須 ≥ configMAX_SYSCALL_INTERRUPT_PRIORITY（即優先權等級要低於或等於此閾值）。
EXTI — 外部中斷/事件控制器

EXTI（External Interrupt/Event Controller） 負責偵測 GPIO 腳位的電位變化並觸發中斷。 STM32F446 有 23 條 EXTI 線，其中 EXTI0–EXTI15 對應 GPIO 的 Pin0–Pin15。 每條 EXTI 線只能連接到一個 GPIO Port（由 SYSCFG_EXTICRx 設定）， 例如 EXTI13 可以連 PA13、PB13、PC13 之一。

c — 設定 PC13（Nucleo 按鍵 B1）觸發 EXTI13 中斷 
/* 1. 啟用 GPIOC 與 SYSCFG 時脈 */
RCC->AHB1ENR |= RCC_AHB1ENR_GPIOCEN;
RCC->APB2ENR |= RCC_APB2ENR_SYSCFGEN;

/* 2. PC13 設為輸入，啟用上拉（按鍵按下接地，放開為高電位）*/
GPIOC->MODER  &= ~(3U << (13 * 2));   /* Input mode */
GPIOC->PUPDR  &= ~(3U << (13 * 2));
GPIOC->PUPDR  |=  (1U << (13 * 2));   /* Pull-up */

/* 3. SYSCFG_EXTICR4：將 EXTI13 映射到 PC（Port C = 0x02）*/
SYSCFG->EXTICR[3] &= ~SYSCFG_EXTICR4_EXTI13;
SYSCFG->EXTICR[3] |=  SYSCFG_EXTICR4_EXTI13_PC;

/* 4. 設定 EXTI13 觸發邊緣（下降緣 = 按鍵按下）*/
EXTI->FTSR |= EXTI_FTSR_TR13;   /* Falling trigger enable */
EXTI->RTSR &= ~EXTI_RTSR_TR13;  /* 不使用上升緣 */

/* 5. 在 EXTI 遮罩暫存器中解除遮罩（允許中斷）*/
EXTI->IMR |= EXTI_IMR_MR13;

/* 6. 在 NVIC 中啟用 EXTI15_10_IRQn，設定優先權 */
NVIC_SetPriority(EXTI15_10_IRQn, 5);
NVIC_EnableIRQ(EXTI15_10_IRQn);

/* 7. ISR — EXTI15_10 涵蓋 Pin 10–15，需判斷來源 */
void EXTI15_10_IRQHandler(void)
{
    if (EXTI->PR & EXTI_PR_PR13) {       /* 確認是 EXTI13 */
        EXTI->PR = EXTI_PR_PR13;          /* 寫 1 清除 Pending flag */
        /* 使用者邏輯 */
        GPIOA->BSRR = (1U << (5 + 16)); /* Toggle LED */
    }
}
必須清除 Pending Flag
ISR 結束前必須清除 EXTI->PR 對應的位元（寫入 1 清除）， 否則 CPU 離開 ISR 後會立刻再次進入，形成無窮迴圈。
2.5 — 理論 + 實作
DMA 控制器
Stream、Channel、傳輸方向與中斷驅動的零 CPU 開銷資料搬移
DMA 基本概念

DMA（Direct Memory Access） 控制器允許在 不佔用 CPU 的情況下， 在記憶體與記憶體、或記憶體與周邊暫存器之間搬移資料。 STM32F446 有兩個 DMA 控制器（DMA1、DMA2），各有 8 條 Stream， 每條 Stream 支援 8 個 Channel（決定連接哪個周邊）。

DMA 傳輸方向 M2M │ Memory → Memory 僅 DMA2 支援 M2P │ Memory → Peripheral 例：記憶體資料送 USART TX P2M │ Peripheral → Memory 例：ADC 結果存入陣列 P2P │ Peripheral → Peripheral 不常用 DMA2 常用 Stream / Channel 對應（部分） Stream 0, Ch 0 │ ADC1 Stream 1, Ch 4 │ USART6 RX Stream 6, Ch 4 │ USART6 TX Stream 7, Ch 5 │ USART1 TX Stream 5, Ch 4 │ USART1 RX
記憶體對記憶體 DMA 搬移範例
c — DMA2 M2M 傳輸 1024 bytes 
#include "stm32f4xx.h"

#define BUF_SIZE  1024

static uint8_t src[BUF_SIZE];
static uint8_t dst[BUF_SIZE];
volatile uint8_t dma_done = 0;

void dma_m2m_start(void)
{
    /* 啟用 DMA2 時脈 */
    RCC->AHB1ENR |= RCC_AHB1ENR_DMA2EN;

    /* 使用 DMA2 Stream 0，Channel 0（M2M 任意 channel 均可）*/
    DMA2_Stream0->CR = 0;             /* 先停用 Stream */
    while (DMA2_Stream0->CR & DMA_SxCR_EN); /* 等待停用完成 */

    DMA2->LIFCR = 0x3FU;              /* 清除 Stream 0 的所有旗標 */

    DMA2_Stream0->PAR  = (uint32_t)src; /* 來源（此模式 PAR 作為來源）*/
    DMA2_Stream0->M0AR = (uint32_t)dst; /* 目的地 */
    DMA2_Stream0->NDTR = BUF_SIZE;

    DMA2_Stream0->CR =
        (0U << DMA_SxCR_CHSEL_Pos) |  /* Channel 0 */
        DMA_SxCR_MINC               |  /* Memory 位址遞增 */
        DMA_SxCR_PINC               |  /* Peripheral（來源）位址遞增 */
        DMA_SxCR_DIR_1              |  /* DIR = 10: M2M */
        DMA_SxCR_TCIE;                 /* Transfer Complete 中斷 */

    NVIC_SetPriority(DMA2_Stream0_IRQn, 3);
    NVIC_EnableIRQ(DMA2_Stream0_IRQn);

    DMA2_Stream0->CR |= DMA_SxCR_EN;  /* 啟動傳輸 */
}

void DMA2_Stream0_IRQHandler(void)
{
    if (DMA2->LISR & DMA_LISR_TCIF0) {
        DMA2->LIFCR = DMA_LIFCR_CTCIF0;  /* 清除 Transfer Complete 旗標 */
        dma_done = 1;
    }
}
循環模式（Circular Mode）與雙緩衝

啟用 DMA_SxCR_CIRC 後，DMA 在完成一次傳輸後會自動從頭開始， 適合連續串流資料（如 ADC 連續取樣、音訊 DMA）。 搭配 雙緩衝模式（Double Buffer Mode） 可以實現 ping-pong 緩衝： DMA 寫入 Buffer A 時，CPU 處理 Buffer B；Buffer A 滿後自動切換， 達到真正的零拷貝、零停頓串流處理。

效能比較
以 115200 baud USART 接收為例：
• 輪詢方式：CPU 必須持續查詢 RXNE 旗標，無法執行其他工作。
• 中斷方式：每收到一個 byte 觸發一次中斷，上下文切換開銷累積顯著。
• DMA 方式：CPU 完全解放，僅在緩衝區滿時接到一次中斷，效率最高。
2.6 — 理論 + 實作
時脈樹與低功耗模式
PLL 配置、時脈來源切換與 Sleep / Stop / Standby 模式
STM32F446 時脈樹

STM32F446RE 的最高系統時脈為 180 MHz。時脈來源有三種： HSI（16 MHz 內部 RC，精度 ±1%）、 HSE（外部晶振，Nucleo 板為 8 MHz）、 LSI / LSE（32 kHz，用於 RTC / IWDG）。 透過 PLL（Phase-Locked Loop）可將 HSI/HSE 倍頻至所需頻率。

PLL 配置（HSE 8 MHz → SYSCLK 180 MHz） HSE (8 MHz) ──▶ ÷ PLLM (8) ──▶ 1 MHz (VCO 輸入) │ ▼ × PLLN (360) 360 MHz (VCO 輸出) │ ┌─────────────────┼───────────────────┐ ▼ ▼ ▼ ÷ PLLP (2) ÷ PLLQ (8) ÷ PLLR (2) SYSCLK 180 MHz USB/SDIO 45 MHz I2S... SYSCLK ──▶ AHB Prescaler (÷1) ──▶ HCLK 180 MHz HCLK ──▶ APB1 Prescaler (÷4) ──▶ PCLK1 45 MHz (APB1 Timer × 2 = 90 MHz) HCLK ──▶ APB2 Prescaler (÷2) ──▶ PCLK2 90 MHz (APB2 Timer × 2 = 180 MHz)
以暫存器設定 PLL 至 180 MHz
c — SystemClock_Config（HSE 8 MHz → 180 MHz） 
void SystemClock_Config(void)
{
    /* 1. 啟用 HSE 並等待穩定 */
    RCC->CR |= RCC_CR_HSEON;
    while (!(RCC->CR & RCC_CR_HSERDY));

    /* 2. 啟用電壓調節器 Scale 1（最高性能，支援 180 MHz）*/
    RCC->APB1ENR |= RCC_APB1ENR_PWREN;
    PWR->CR |= PWR_CR_VOS;  /* Scale 1 */

    /* 3. 先設定 Flash 等待狀態（180 MHz 需 5 WS）*/
    FLASH->ACR = FLASH_ACR_LATENCY_5WS |
                 FLASH_ACR_ICEN        |   /* 指令快取 */
                 FLASH_ACR_DCEN        |   /* 資料快取 */
                 FLASH_ACR_PRFTEN;         /* 預取 */

    /* 4. 設定 PLL：HSE / 8 × 360 / 2 = 180 MHz */
    RCC->PLLCFGR = (8U   << RCC_PLLCFGR_PLLM_Pos) |   /* M = 8  */
                   (360U << RCC_PLLCFGR_PLLN_Pos) |   /* N = 360 */
                   (0U   << RCC_PLLCFGR_PLLP_Pos) |   /* P = 2 (00) */
                   RCC_PLLCFGR_PLLSRC_HSE           |   /* 來源 HSE */
                   (8U   << RCC_PLLCFGR_PLLQ_Pos);     /* Q = 8 → 45 MHz USB */

    /* 5. 啟用 PLL 並等待鎖定 */
    RCC->CR |= RCC_CR_PLLON;
    while (!(RCC->CR & RCC_CR_PLLRDY));

    /* 6. 設定 AHB/APB 預分頻器 */
    RCC->CFGR = RCC_CFGR_HPRE_DIV1  |   /* AHB = 180 MHz */
                RCC_CFGR_PPRE1_DIV4 |   /* APB1 = 45 MHz */
                RCC_CFGR_PPRE2_DIV2;    /* APB2 = 90 MHz */

    /* 7. 切換 SYSCLK 來源為 PLL */
    RCC->CFGR |= RCC_CFGR_SW_PLL;
    while ((RCC->CFGR & RCC_CFGR_SWS) != RCC_CFGR_SWS_PLL);
}
低功耗模式比較
模式 進入方式 喚醒來源 SRAM / 暫存器 典型電流
Sleep WFI / WFE 任何中斷 保留 ~14 mA
Stop PWR_EnterSTOPMode() EXTI、RTC Alarm、IWDG 保留 ~1 mA
Standby PWR_EnterSTANDBYMode() WKUP pin、RTC、IWDG、Reset 喪失（備份域保留） ~2.4 µA
Stop 模式喚醒後的時脈
從 Stop 模式喚醒後，系統時脈自動切回 HSI（16 MHz），PLL 狀態不保留。 若應用需要以 180 MHz 繼續執行，喚醒後必須重新執行 SystemClock_Config()。
