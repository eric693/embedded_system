CHAPTER 01

嵌入式系統概論

建立對嵌入式世界的完整認識，了解與通用電腦的本質差異

Lesson 1.1
什麼是嵌入式系統？定義與特性

嵌入式系統（Embedded System）是指為了特定任務而設計的計算系統，通常嵌入在更大型的裝置或設備之中。與桌上型電腦不同，嵌入式系統必須在嚴格的資源限制下可靠運行，這些限制包括記憶體大小、運算速度、功耗，以及即時回應的時間要求。

判斷一個系統是否屬於嵌入式系統，可從三個維度觀察：功能專一性（只做特定幾件事）、資源受限性（RAM 可能只有數 KB）、以及可靠性要求（汽車 ABS 系統不能出錯）。

「即時性」並不等於「快」。即時系統（Real-time System）的核心要求是在確定的時間限制內完成任務，無論這個時限是 1 毫秒還是 1 秒。

Lesson 1.2
嵌入式系統的應用領域

嵌入式系統幾乎無所不在。消費性電子（智慧家電、穿戴裝置）、汽車電子（ECU、ADAS）、工業控制（PLC、感測器節點）、醫療設備（心律調整器、血糖機）、航太與國防，都是嵌入式技術的舞台。

醫療輸液幫浦、心電圖機IEC 62304 軟體生命週期

Lesson 1.3
常見處理器架構概覽

選擇正確的處理器架構是嵌入式設計的第一步。目前主流架構有三大家族：

ARM Cortex-M 系列（M0/M3/M4/M7/M33）是 32 位元 MCU 的業界主流，STM32、nRF52、RP2040 等都屬此家族。Cortex-A 系列則用於需要跑 Linux 的應用處理器（如 Raspberry Pi）。

AVR（Atmel，現 Microchip）是 8 位元架構，Arduino Uno 的 ATmega328P 即屬此類，適合學習入門與低成本應用。

RISC-V 是開放指令集架構，近年在 IoT 領域快速崛起，GD32VF103 是代表性 MCU。

本課程以 ARM Cortex-M（STM32F4 系列） 為主要實作平台，使用 HAL 庫與 STM32CubeIDE 開發環境，搭配 Raspberry Pi 4 進行嵌入式 Linux 實驗。

微控制器硬體基礎

Lesson 2.1
記憶體架構：Flash、SRAM 與記憶體映射

STM32F407 的記憶體空間採用哈佛架構變體，程式碼（Flash）與資料（SRAM）位於不同的實體記憶體，但透過統一的 32 位元位址空間存取。理解位址映射是讀懂 Linker Script 的前提。

記憶體起始位址大小（F407）用途
Flash0x080000001 MB程式碼、唯讀常數
SRAM10x20000000112 KB堆疊、堆積、全域變數
SRAM20x2001C00016 KB備用 RAM
CCM RAM0x1000000064 KB僅 CPU 可存取，速度最快
Peripheral0x40000000—記憶體映射暫存器
c — 直接操作記憶體映射暫存器（不用 HAL） 
/* 透過記憶體映射位址直接控制 GPIOA */
#define GPIOA_BASE   0x40020000UL
#define GPIOA_MODER  (*(volatile uint32_t *)(GPIOA_BASE + 0x00))
#define GPIOA_ODR    (*(volatile uint32_t *)(GPIOA_BASE + 0x14))

void gpio_init(void) {
    /* 設定 PA5 為輸出模式（MODER[11:10] = 01）*/
    GPIOA_MODER &= ~(0x3 << 10);
    GPIOA_MODER |=  (0x1 << 10);
}

void gpio_toggle(void) {
    GPIOA_ODR ^= (1 << 5);  /* 切換 PA5 */
}
Lesson 2.2
時脈系統：RCC 與 PLL 配置

STM32 的時脈樹（Clock Tree）決定了整個系統的運行速度。從外部晶振（HSE）或內部 RC 振盪器（HSI）出發，經過 PLL 倍頻後得到系統時脈（SYSCLK），再分頻給 AHB、APB1、APB2 匯流排。

c — 配置 PLL 使 SYSCLK = 168 MHz（HSE = 8 MHz） 
/*
 * PLL 公式：
 *   VCO 輸入  = HSE / PLLM  = 8 / 4 = 2 MHz
 *   VCO 輸出  = VCO 輸入 × PLLN = 2 × 168 = 336 MHz
 *   SYSCLK    = VCO 輸出 / PLLP  = 336 / 2 = 168 MHz
 *   USB/SDIO  = VCO 輸出 / PLLQ  = 336 / 7 = 48 MHz
 */

RCC_OscInitTypeDef osc = {0};
osc.OscillatorType      = RCC_OSCILLATORTYPE_HSE;
osc.HSEState            = RCC_HSE_ON;
osc.PLL.PLLState        = RCC_PLL_ON;
osc.PLL.PLLSource       = RCC_PLLSOURCE_HSE;
osc.PLL.PLLM            = 4;
osc.PLL.PLLN            = 168;
osc.PLL.PLLP            = RCC_PLLP_DIV2;
osc.PLL.PLLQ            = 7;
HAL_RCC_OscConfig(&osc);

RCC_ClkInitTypeDef clk = {0};
clk.ClockType      = RCC_CLOCKTYPE_SYSCLK
                   | RCC_CLOCKTYPE_HCLK
                   | RCC_CLOCKTYPE_PCLK1
                   | RCC_CLOCKTYPE_PCLK2;
clk.SYSCLKSource   = RCC_SYSCLKSOURCE_PLLCLK;
clk.AHBCLKDivider  = RCC_SYSCLK_DIV1;   /* AHB  = 168 MHz */
clk.APB1CLKDivider = RCC_HCLK_DIV4;     /* APB1 = 42 MHz  */
clk.APB2CLKDivider = RCC_HCLK_DIV2;     /* APB2 = 84 MHz  */
HAL_RCC_ClockConfig(&clk, FLASH_LATENCY_5);
提高時脈頻率時，必須同步調整 Flash 存取延遲（FLASH_LATENCY）。在 3.3V 供電下，168 MHz 需要設定 5 個等待週期（WS=5）。
Lesson 2.3
中斷系統：NVIC、IRQ 與優先權

ARM Cortex-M 的嵌套向量中斷控制器（NVIC）支援最多 240 個外部中斷，並允許中斷巢狀（高優先權中斷可打斷低優先權中斷的處理程式）。STM32F4 使用 4 位元優先權，分為搶佔優先權（Preemption Priority）與子優先權（Sub Priority）。

c — EXTI 外部中斷配置與 ISR 撰寫 
/* 配置 PA0 的下降緣觸發中斷，搶佔優先權 = 1，子優先權 = 0 */
GPIO_InitTypeDef gpio = {0};
gpio.Pin  = GPIO_PIN_0;
gpio.Mode = GPIO_MODE_IT_FALLING;  /* 下降緣觸發 */
gpio.Pull = GPIO_PULLUP;
HAL_GPIO_Init(GPIOA, &gpio);

HAL_NVIC_SetPriority(EXTI0_IRQn, 1, 0);
HAL_NVIC_EnableIRQ(EXTI0_IRQn);

/* ISR：名稱必須與向量表一致 */
void EXTI0_IRQHandler(void) {
    HAL_GPIO_EXTI_IRQHandler(GPIO_PIN_0);
}

/* HAL Callback，在 ISR 清旗標後呼叫 */
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    if (GPIO_Pin == GPIO_PIN_0) {
        /* 按鍵事件處理：盡量短，不要在 ISR 內做複雜運算 */
        button_event_flag = 1;
    }
}
ISR 應盡可能短暫。複雜的邏輯應透過旗標（flag）或 RTOS 的 Queue 傳遞給主迴圈或工作任務處理，避免 ISR 佔用 CPU 時間過長。
Lesson 2.4
DMA：直接記憶體存取

DMA（Direct Memory Access）允許周邊裝置直接讀寫記憶體，不需要 CPU 介入每一個位元組的搬移。這對 ADC 連續取樣、UART 大量傳輸、SPI 顯示器驅動等場景至關重要，可大幅降低 CPU 佔用率。

c — UART 使用 DMA 接收，非阻塞模式 
uint8_t rx_buf[64];

/* 啟動 DMA 接收 64 位元組，完成後觸發回呼 */
HAL_UART_Receive_DMA(&huart2, rx_buf, sizeof(rx_buf));

/* DMA 接收完成回呼 */
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    if (huart->Instance == USART2) {
        /* rx_buf 已填滿，可以處理資料 */
        process_data(rx_buf, sizeof(rx_buf));
        /* 重新啟動下一次接收 */
        HAL_UART_Receive_DMA(&huart2, rx_buf, sizeof(rx_buf));
    }
}

Lesson 3.1
位元運算與暫存器操作

在嵌入式開發中，位元運算是日常。設定、清除、切換、讀取特定位元是操作硬體暫存器的基本功。

c — 常用位元操作模式 
#define BIT(n)           (1U << (n))
#define SET_BIT(reg, n)   ((reg) |=  BIT(n))
#define CLR_BIT(reg, n)   ((reg) &= ~BIT(n))
#define TGL_BIT(reg, n)   ((reg) ^=  BIT(n))
#define READ_BIT(reg, n)  (((reg) >> (n)) & 1U)

/* 設定多位元欄位（先清除再設值）*/
#define SET_FIELD(reg, mask, shift, val) \
    ((reg) = ((reg) & ~(mask)) | (((val) << (shift)) & (mask)))

/* 範例：設定 USART_CR1 的 OVER8 位元（bit 15）*/
SET_BIT(USART2->CR1, 15);

/* 範例：設定 GPIO OSPEEDR[3:2] = 0b10（High Speed）*/
SET_FIELD(GPIOA->OSPEEDR, 0x3 << 2, 2, 0x2);
Lesson 3.2
volatile、const 與型別修飾詞

volatile 告訴編譯器：這個變數的值可能在程式的控制流之外被改變（例如 ISR、硬體暫存器），因此每次存取都必須真正讀寫記憶體，不能被最佳化快取到暫存器中。

c — volatile 的正確使用情境 
/* 錯誤：編譯器可能最佳化成空迴圈，認為 flag 永遠是 0 */
int flag = 0;
while (flag == 0) { }  /* ISR 中設定了 flag，但這裡看不到 */

/* 正確：volatile 確保每次迴圈都真正讀取 flag */
volatile int flag = 0;
while (flag == 0) { }

/* 常數暫存器：volatile const — 只能讀，但每次都要讀 */
volatile const uint32_t *status_reg =
    (volatile const uint32_t *)0x40020010UL;
Lesson 3.3
連結腳本（Linker Script）解析

Linker Script（.ld 檔）告訴連結器如何將各個目的檔的段（section）放置到正確的記憶體位址。理解它是閱讀啟動程式（startup.s）與優化記憶體佈局的關鍵。

ld — STM32F407 簡化版 Linker Script 
/* 定義記憶體區域 */
MEMORY {
    FLASH (rx)  : ORIGIN = 0x08000000, LENGTH = 1024K
    SRAM  (rwx) : ORIGIN = 0x20000000, LENGTH = 128K
}

SECTIONS {
    /* 向量表與程式碼放入 Flash */
    .isr_vector : { KEEP(*(.isr_vector)) } > FLASH
    .text       : { *(.text*) *(.rodata*) } > FLASH

    /* 初始化資料：儲存在 Flash，執行期複製到 SRAM */
    .data : {
        _sdata = .;
        *(.data*)
        _edata = .;
    } > SRAM AT > FLASH      /* VMA=SRAM, LMA=FLASH */

    /* 未初始化資料（啟動時清零）*/
    .bss : {
        _sbss = .;
        *(.bss*)
        *(COMMON)
        _ebss = .;
    } > SRAM

    /* 堆疊放在 SRAM 最頂端 */
    _estack = ORIGIN(SRAM) + LENGTH(SRAM);
}

Lesson 4.1
UART 串列通訊

UART（Universal Asynchronous Receiver-Transmitter）是最常用的非同步串列通訊介面。它只需要兩條線（TX、RX），不需要時脈線，靠雙方預先約定好鮑率（Baud Rate）來同步資料。

參數典型值說明
Baud Rate115200每秒傳輸的符號數
Data bits8每幀的資料位元數
Stop bits1停止位元數
ParityNone奇偶校驗
c — UART printf 重定向至 ST-Link Virtual COM 
/* 重載 _write()，讓 printf 輸出到 UART2 */
int _write(int fd, char *buf, int count) {
    HAL_UART_Transmit(&huart2,
                      (uint8_t *)buf,
                      (uint16_t)count,
                      HAL_MAX_DELAY);
    return count;
}

/* 之後即可直接使用 printf */
printf("Temperature: %.2f C\r\n", temperature);
Lesson 4.2
SPI 協定：主從架構與時序

SPI（Serial Peripheral Interface）是同步全雙工協定，四線制：SCLK（時脈）、MOSI（主傳從收）、MISO（從傳主收）、CS（片選）。時序模式由 CPOL（時脈極性）與 CPHA（時脈相位）決定，共有 Mode 0～3 四種組合。

c — SPI 讀取 BME280 感測器暫存器 
/* BME280 SPI 讀取：最高位元為 1 表示讀取 */
uint8_t bme280_read_reg(uint8_t reg) {
    uint8_t tx[2] = { reg | 0x80, 0xFF };
    uint8_t rx[2] = { 0 };

    HAL_GPIO_WritePin(SPI_CS_GPIO_Port,
                      SPI_CS_Pin,
                      GPIO_PIN_RESET);  /* CS 拉低：選中裝置 */

    HAL_SPI_TransmitReceive(&hspi1, tx, rx, 2, 10);

    HAL_GPIO_WritePin(SPI_CS_GPIO_Port,
                      SPI_CS_Pin,
                      GPIO_PIN_SET);    /* CS 拉高：釋放裝置 */
    return rx[1];
}
Lesson 4.3
I²C 協定：位址機制與多裝置匯流排

I²C 只需兩條線（SDA、SCL），支援多主多從架構。每個裝置有 7 位元（或 10 位元）位址，主裝置透過發送啟始條件（START）、位址幀、讀寫位元來選擇從裝置。

c — I²C 讀取 MPU-6050 加速度計（7 位元位址 = 0x68） 
#define MPU6050_ADDR  (0x68 << 1)  /* HAL 使用 8 位元格式 */
#define ACCEL_XOUT_H  0x3B

int16_t mpu6050_read_accel_x(void) {
    uint8_t reg = ACCEL_XOUT_H;
    uint8_t buf[2];

    /* 寫入要讀取的暫存器位址 */
    HAL_I2C_Master_Transmit(&hi2c1, MPU6050_ADDR,
                             &reg, 1, 10);

    /* 連續讀取 2 個位元組（高位元組 + 低位元組）*/
    HAL_I2C_Master_Receive(&hi2c1, MPU6050_ADDR,
                            buf, 2, 10);

    /* 合併成 16 位元有號整數 */
    return (int16_t)((buf[0] << 8) | buf[1]);
}
Lesson 4.4
ADC 取樣與 PWM 輸出

ADC（Analog-to-Digital Converter）將類比電壓轉換為數位值。STM32F407 內建 12 位元 ADC（0～4095 對應 0～3.3V）。PWM（Pulse Width Modulation）則透過調整方波的占空比（Duty Cycle）來等效控制輸出電壓，廣泛用於馬達速度控制與 LED 亮度調節。

c — ADC 單次轉換 + TIM3 PWM 輸出 
/* ADC 單次轉換，讀取 PA1（Channel 1）*/
uint32_t adc_read(void) {
    HAL_ADC_Start(&hadc1);
    HAL_ADC_PollForConversion(&hadc1, 10);
    return HAL_ADC_GetValue(&hadc1);
}

/* PWM：TIM3 CH1，將 ADC 值映射到 0~100% 占空比 */
void pwm_set_duty(uint32_t adc_val) {
    /* ADC 12 位元（0~4095）→ TIM3 ARR（假設 = 999）*/
    uint32_t duty = (adc_val * 1000) / 4095;
    __HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_1, duty);
}

/* 主迴圈：光敏電阻控制 LED 亮度 */
HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_1);
while (1) {
    pwm_set_duty(adc_read());
    HAL_Delay(10);
}
CHAPTER 05

即時作業系統（RTOS）

FreeRTOS 多工設計、排程策略與任務間同步機制

Lesson 5.1
RTOS vs 裸機：何時需要 RTOS？

裸機（Bare-metal）開發使用 super-loop 或狀態機驅動，適合邏輯簡單、任務間耦合低的應用。當系統需要同時處理多個具有不同時間要求的任務，或任務間需要複雜的同步與通訊時，RTOS 能提供清晰的架構。

考量裸機 Super-loopRTOS
確定性難以保證可設定任務優先權
程式碼複雜度低（初期）需學習 API
RAM 開銷極低每個任務需獨立堆疊
除錯難度較直觀需工具支援（trace）
適用場景單一功能感測器多任務 IoT 閘道器
Lesson 5.2
FreeRTOS 任務建立與排程

FreeRTOS 使用搶佔式優先權排程（Preemptive Priority Scheduling）。數字越大優先權越高。排程器每隔一個 Tick（通常 1ms）判斷是否要切換到更高優先權的就緒任務。

c — 建立兩個優先權不同的任務 
void sensor_task(void *param) {
    for (;;) {
        float temp = bme280_read_temperature();
        printf("Temp: %.1f\r\n", temp);
        vTaskDelay(pdMS_TO_TICKS(500));  /* 讓出 CPU 500ms */
    }
}

void comm_task(void *param) {
    for (;;) {
        mqtt_publish_pending();
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

int main(void) {
    HAL_Init();
    SystemClock_Config();

    xTaskCreate(sensor_task, "Sensor",
                 256,    /* 堆疊大小（字組數）*/
                 NULL,
                 2,      /* 優先權（較高）*/
                 NULL);

    xTaskCreate(comm_task, "Comm",
                 512, NULL, 1, NULL);

    vTaskStartScheduler();  /* 永不返回 */
    while (1);
}
Lesson 5.3
Queue、Semaphore 與 Mutex

任務間不能直接共享全域變數（競態條件），需使用 RTOS 提供的同步原語。Queue 用於傳遞資料；Semaphore 用於計數或信號；Mutex 用於保護共享資源（含優先權繼承機制）。

c — Queue：感測器任務傳資料給通訊任務 
QueueHandle_t data_queue;

typedef struct {
    float    temperature;
    float    humidity;
    uint32_t timestamp;
} SensorData_t;

/* 建立可存放 8 筆資料的 Queue */
data_queue = xQueueCreate(8, sizeof(SensorData_t));

/* 生產者任務 */
void sensor_task(void *p) {
    SensorData_t d;
    for (;;) {
        d.temperature = bme280_read_temperature();
        d.humidity    = bme280_read_humidity();
        d.timestamp   = HAL_GetTick();
        xQueueSend(data_queue, &d, 0);
        vTaskDelay(pdMS_TO_TICKS(500));
    }
}

/* 消費者任務 */
void comm_task(void *p) {
    SensorData_t d;
    for (;;) {
        /* 阻塞等待，直到 Queue 有資料 */
        if (xQueueReceive(data_queue, &d, portMAX_DELAY)) {
            mqtt_publish(d.temperature, d.humidity);
        }
    }
}
使用 Mutex 而非 Semaphore 來保護共享資源，原因是 Mutex 具有優先權繼承機制，可以防止優先權反轉（Priority Inversion）問題。

Lesson 6.3
字元設備驅動程式開發

Linux 驅動程式是核心模組（.ko）的一種。字元設備驅動實作 file_operations 結構，提供 open、read、write、ioctl 等操作，讓使用者空間程式透過 /dev/xxx 與硬體互動。

c — 最小字元設備驅動程式骨架 
#include <linux/module.h>
#include <linux/fs.h>
#include <linux/uaccess.h>

static int major;
static char kbuf[256];

static ssize_t mydev_read(struct file *f,
                            char __user *buf,
                            size_t len, loff_t *off) {
    size_t n = min(len, sizeof(kbuf));
    if (copy_to_user(buf, kbuf, n))
        return -EFAULT;
    return n;
}

static const struct file_operations fops = {
    .owner = THIS_MODULE,
    .read  = mydev_read,
};

static int __init mydev_init(void) {
    major = register_chrdev(0, "mydev", &fops);
    pr_info("mydev: registered with major %d\n", major);
    return 0;
}

static void __exit mydev_exit(void) {
    unregister_chrdev(major, "mydev");
}

module_init(mydev_init);
module_exit(mydev_exit);
MODULE_LICENSE("GPL");
bash — 編譯並載入核心模組

Lesson 7.1
STM32 低功耗模式

STM32 提供多個層級的低功耗模式，功耗由高到低為：Run → Sleep → Stop → Standby。選擇哪個模式取決於需要保留的上下文與喚醒來源。

模式CPUSRAM典型電流喚醒來源
Run執行保留~50 mA—
Sleep停止保留~1 mA任何中斷
Stop 2停止保留~2 µAEXTI、RTC
Standby停止清除~0.3 µAWKUP Pin、RTC
c — 進入 Stop 2 模式，RTC 鬧鐘喚醒 
/* 設定 RTC 鬧鐘在 30 秒後觸發 */
RTC_AlarmTypeDef alarm = {0};
HAL_RTC_GetTime(&hrtc, &alarm.AlarmTime, RTC_FORMAT_BIN);
alarm.AlarmTime.Seconds =
    (alarm.AlarmTime.Seconds + 30) % 60;
alarm.AlarmMask         = RTC_ALARMMASK_ALL;
alarm.AlarmMask        &= ~RTC_ALARMMASK_SECONDS;
alarm.Alarm             = RTC_ALARM_A;
HAL_RTC_SetAlarm_IT(&hrtc, &alarm, RTC_FORMAT_BIN);

/* 關閉不需要的周邊節省電力 */

/* 進入 Stop 2 模式（最低功耗 Stop，保留 SRAM）*/
HAL_PWREx_EnterSTOP2Mode(PWR_STOPENTRY_WFI);

SystemClock_Config();

Lesson 7.2
OTA 韌體更新：Dual Bank Bootloader

OTA（Over-The-Air）更新讓嵌入式裝置可以在不拆機的情況下更新韌體。Dual Bank 設計將 Flash 分成兩個區域（Bank A / Bank B），新韌體寫入非執行中的 Bank，驗證後再切換，確保更新失敗時可以回滾。

c — Bootloader 韌體版本驗證與跳轉邏輯 
#define APP_ADDR       0x08020000UL  /* Bank B 起始位址 */
#define MAGIC_WORD     0xDEADBEEFUL  /* 韌體有效標記 */

typedef struct {
    uint32_t magic;
    uint32_t version;
    uint32_t crc32;
    uint32_t size;
} FirmwareHeader_t;

void bootloader_jump_to_app(void) {
    FirmwareHeader_t *hdr =
        (FirmwareHeader_t *)APP_ADDR;

    /* 驗證魔術字 */
    if (hdr->magic != MAGIC_WORD) {
        Error_Handler();
        return;
    }

    /* 驗證 CRC */
    uint32_t calc = crc32_compute(
        (uint8_t *)(APP_ADDR + sizeof(FirmwareHeader_t)),
        hdr->size);
    if (calc != hdr->crc32) {
        Error_Handler();
        return;
    }

    /* 關閉所有中斷，跳轉到應用程式 */
    __disable_irq();
    uint32_t app_stack = *(uint32_t *)APP_ADDR;
    uint32_t app_reset = *(uint32_t *)(APP_ADDR + 4);
    __set_MSP(app_stack);
    ((void (*)(void))app_reset)();
}
Lesson 7.3
Watchdog 定時器與故障安全

Watchdog（看門狗）是一個獨立運行的計時器，若主程式在期限內沒有「餵狗」（重置計數器），Watchdog 就會觸發系統重置，防止程式跑飛或卡死。STM32 提供 IWDG（獨立看門狗，使用獨立 LSI 時脈）與 WWDG（視窗看門狗）兩種。

c — IWDG 配置：1 秒超時，主迴圈定期餵狗 
/*
 * IWDG 時脈 = LSI / Prescaler / Reload
 * LSI ≈ 32 kHz，Prescaler = /256，Reload = 125
 * 超時 = 125 / (32000/256) = 1.0 秒
 */
hiwdg.Instance       = IWDG;
hiwdg.Init.Prescaler = IWDG_PRESCALER_256;
hiwdg.Init.Reload    = 125;
HAL_IWDG_Init(&hiwdg);

/* 主迴圈每次迭代都要餵狗 */
while (1) {
    do_sensor_reading();       /* 任何一步卡住超過 1 秒 */
    do_data_processing();      /* 系統就會自動重置      */
    do_network_transmit();
    HAL_IWDG_Refresh(&hiwdg);  /* 餵狗：重置計數器      */
}
IWDG 一旦啟動便無法關閉（即使在偵錯模式下仍然運行）。若在偵錯時頻繁碰到 Watchdog 重置，可在 STM32CubeIDE 的 DBGMCU 暫存器中設定 DBG_IWDG_STOP，讓 Watchdog 在偵錯暫停時一併暫停。

c — SensorTask 核心邏輯 
void SensorTask(void *arg) {
    EnvData_t data;
    for (;;) {
        data.temperature = bme280_get_temperature();
        data.humidity    = bme280_get_humidity();
        data.pressure    = bme280_get_pressure();
        data.aqi         = mq135_get_aqi();
        data.tick        = xTaskGetTickCount();

        /* 送入 Queue，不等待（若滿則丟棄最舊資料）*/
        if (xQueueSend(env_queue, &data, 0) != pdTRUE) {
            EnvData_t discard;
            xQueueReceive(env_queue, &discard, 0);
            xQueueSend(env_queue, &data, 0);
        }
        vTaskDelay(pdMS_TO_TICKS(5000));
    }
}

void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan) {

import json, time, smbus2

BROKER  = "mqtt.example.com"
PORT    = 8883   # TLS
TOPIC   = "gateway/env/data"
CERT    = "/etc/certs/client.crt"
KEY     = "/etc/certs/client.key"
CA      = "/etc/certs/ca.crt"

client = mqtt.Client(client_id="rpi-gw-001")
client.tls_set(ca_certs=CA, certfile=CERT, keyfile=KEY)
client.connect(BROKER, PORT, keepalive=60)
client.loop_start()

bus = smbus2.SMBus(1)

while True:
    raw = bus.read_i2c_block_data(0x76, 0xF7, 8)
    temp = bme280_compensate_temperature(raw)

    payload = json.dumps({
        "device_id": "rpi-gw-001",
        "temperature": round(temp, 2),
        "ts": int(time.time())
    })

    client.publish(TOPIC, payload, qos=1)
    time.sleep(10)
